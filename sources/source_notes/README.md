# Source Notes

Add one note per ingested source document.

Each note should include:

- source ID,
- source title,
- ingestion date,
- source version or URL,
- thesis,
- mechanisms,
- evidence,
- failure modes,
- book chapters supported,
- unresolved questions.

Do not summarize a source unless the source text was actually available in the working context.

`backbone_manifest.json` lists the source notes that must exist before a full-book drafting run starts. These notes are context-loading artifacts. They do not by themselves promote any book claim above `argument`; promotion still requires an explicit claim mapping in Appendix C and, where relevant, proof, code, test, or external-literature artifacts.
