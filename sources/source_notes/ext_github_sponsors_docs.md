## Thesis

GitHub Sponsors is relevant to Artifact Steward Agents because it provides a GitHub-native funding surface for open-source contributors and organizations. It can inform steward funding intake, contributor support, and public project sustainability records, but it does not prove that a steward should allocate money or that sponsorship incentives are governance-safe.

## Mechanisms

- Let eligible open-source contributors and organizations receive sponsorship through GitHub.
- Recognize multiple contribution categories, including code, documentation, issue triage, leadership, project management, mentorship, design, and related open-source work.
- Provide sponsor-profile and organization setup flows.
- Connect funding support to the repository and contributor context where project work already occurs.
- Offer a near-term substrate for human-approved funding and supporter records.

## Evidence

- The official GitHub documentation describes GitHub Sponsors eligibility and contribution categories for sponsored developers and organizations.
- This repository has not configured GitHub Sponsors, received sponsorships, created payout records, or connected sponsorship to steward policies.
- Use this source for sponsorship-surface vocabulary only.

## Failure Modes

- Sponsorship does not prove that a project is sustainable, safe, high-quality, or mission-aligned.
- Funding signals can bias roadmap priorities if separated contribution, governance, and evidence ledgers are absent.
- Contributor eligibility and region support are platform constraints, not steward authority.
- A steward should not treat sponsor money as automatic permission to spend, publish, merge, or change governance.
