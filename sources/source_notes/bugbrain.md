## Thesis

BugBrain is an embedded, text-native AGI MVP blueprint centered on a bare-metal Rust microkernel and an evolved fractal spiking language organism. For this book, its useful contribution is not the consciousness claim, but the disciplined attempt to make tiny, local, resource-constrained cognition explicit enough for build, flash, bridge, train, and test workflows.

## Mechanisms

- Boot directly into a text-native microkernel loop with UART input and output.
- Use fractal spiking nodes, mutation, pruning, valence, entropy tracking, and user feedback as a lightweight adaptive mechanism.
- Pre-train or "birth" the state in QEMU before flashing a target device.
- Provide training, flashing, and bridge tools around the embedded runtime.
- Treat user feedback as a continual-learning signal after deployment.
- Page fractal layers from storage to fit constrained hardware.

## Evidence

- The source is a technical guide and code-oriented blueprint with substantial pseudocode and implementation skeletons.
- The repository has not built, flashed, emulated, benchmarked, or validated BugBrain.
- Consciousness, AGI, economic usefulness, and hardware performance claims remain speculative/source-proposed.

## Failure Modes

- Treating proto-consciousness vocabulary as evidence of consciousness.
- Assuming pseudocode compiles or behaves as intended on target hardware.
- Letting user feedback optimize personalization without safety and regression gates.
- Confusing resource-constrained design with proven efficiency.
- Omitting adversarial tests for generated outputs, feedback poisoning, and unsafe persistence.
