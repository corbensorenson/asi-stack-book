## Thesis

Ratcheting Modular Intelligence, or RMI, argues that AI systems should improve by turning development pressure into verified modular structure. The source combines compact generative structure, active compression, cognitive loop closure, benchmark ratcheting, and octopus-style routing into one growth process. The central design rule is that the frontier should move while the floor remains protected by regression, residual escrow, and lifecycle discipline.

## Mechanisms

- Treat capability growth as a loop: benchmark frontier, routed specialist attempts, logs, loop closure, residual escrow, benchmark diagnosis, independent arm/router improvement, regression preservation, and harder frontier selection.
- Use a head/router, specialist arms, arm registry, memory router, permission router, tool registry, benchmark ledger, residual escrow ledger, and verification/safety layer.
- Preserve three execution modes: interpreter mode for novel tasks, compiled-tool mode for closed recurring tasks, and reflex/failsafe mode for safety-critical cases.
- Use mastery thresholds, time-decayed thresholds for non-safety-critical work, critical-failure vetoes, and residual escrow instead of pretending every benchmark remainder is either solved or irrelevant.
- Add, split, merge, retire, and ratchet specialist arms based on residuals, bloat, redundancy, staleness, and benchmark pressure.

## Evidence

- The source is a conceptual framework and AI systems architecture proposal.
- It contains source-reported references to benchmark saturation, mixture-of-experts, tool-use, skill-library, runtime-verification, and distributed-control literature.
- It provides design logic, failure-mode analysis, ledgers, lifecycle states, metrics, and an implementation roadmap.
- No independent reproduction, benchmark run, prototype inspection, or Lean proof was performed in this repo as part of this note.

## Failure Modes

- Benchmark gaming and leaderboard addiction.
- Tail obsession, tail erasure, and residual hiding.
- Tool bloat, arm bloat, bad routing, and router monolith drift.
- Under-quarantine, over-quarantine, reflex gaps, and architecture churn.
- Treating public benchmarks as the only internal development target.
