## Thesis

Circle Calculus is a proof-carrying finite cyclic mathematics project that ties Lean proofs, Python reference models, Rust utilities, theorem manifests, papers, and a Quarto living book to explicit proof status. For the ASI Stack, its value is not "circles everywhere"; its value is a disciplined proof boundary for cyclic addressing, phase, recurrence, coverage, and contract receipts.

## Mechanisms

- Use Lean declarations and theorem manifests as the proof authority.
- Use Python reference models, CLI receipts, JSON schemas, generated contract packs, and sidecar results as evidence layers rather than proofs.
- Distinguish public contract families from downstream compatibility/transfer contract families.
- Provide stable entry points for finite orbit/period logic, AI contracts, RoPE receipts, and batch contract checks.
- Define a proof boundary: a claim is proved only when it has a theorem id, compiled Lean declaration, manifest status, passing Lean build, and no forbidden proof placeholders.
- Keep performance, model-quality, deployment-safety, and universal-compression claims outside the theorem layer.

## Evidence

- The repository README describes proof policy, stable APIs, verification commands, and repository layout.
- It names Lean, Python, Rust, manifests, papers, sidecars, tests, and Quarto site surfaces.
- The source states that AI/system contracts prove finite structural facts such as cyclic indexing, RoPE phase-bank conditions, sparse-attention coverage, ring-buffer freshness, recurrence schedules, and circulant/block-cyclic mixer laws.
- The Circle repository was not built or tested from this ASI Stack repo as part of this note.

## Failure Modes

- Treating Python tests, diagrams, widgets, benchmarks, or generated JSON as Lean proofs.
- Treating finite structural contracts as evidence of model quality, speed, memory improvement, longer context, deployment safety, or universal compression.
- Importing Circle concepts into the ASI Stack without preserving theorem ids, non-claims, and proof status.
- Letting mathematical elegance outrun hardware, baseline, or workload evidence.
