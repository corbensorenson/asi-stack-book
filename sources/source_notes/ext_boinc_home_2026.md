## Thesis

BOINC is the strongest simple precedent for volunteer compute: user machines can download jobs and run background scientific work. The hive chapter should treat this as an existence pattern for donated capacity, not as a governed personal AI substrate.

## Mechanisms

- Let participating computers download scientific computing jobs.
- Run jobs in the background for selected science projects.
- Aggregate volunteer machine capacity across many users.
- Give scientists a project framework for distributed computation.

## Evidence

- The official BOINC site describes the app downloading scientific jobs and running them in the background on volunteer computers.
- This repository has not joined BOINC, run a job, inspected BOINC sandboxing, or measured reliability.
- Use this source to ground volunteer-compute lineage and to distinguish public compute donation from private hive authority.

## Failure Modes

- Volunteer compute can normalize background work without adequate user visibility.
- Scientific workload trust does not transfer to arbitrary AI or repository work.
- Background execution must not become hidden spending, surveillance, or private-data exposure.
