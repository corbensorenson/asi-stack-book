## Thesis

Compact Generative Systems, or CGS, argues that powerful systems often use a compact, structured, verifiable core to reconstruct, predict, generate, control, coordinate, or govern something larger than itself. The source's stronger claim is not merely that compression matters; it is that compression becomes most useful when paired with generation, residual accounting, verification, and governance.

## Mechanisms

- Model a CGS as a tuple of seed, rule system, memory/state, residual/error, verification, and governance/generation interface.
- Ask for the smallest adequate structure that can generate or govern the target without hiding unacceptable residual complexity.
- Evaluate quality through generative leverage, fidelity, residual burden, verification cost, governance power, and hidden complexity debt.
- Use a design template covering target system, compact seed, rule system, memory/state, residual channel, correction mechanism, verification contract, governance interface, cost accounting, and failure analysis.
- Apply the frame to AI agents, software architecture, scientific theories, compression systems, controllers, organizational systems, and morphology.

## Evidence

- The source is a conceptual framework and research program.
- It contains source-reported links to MDL, algorithmic information theory, Solomonoff induction, compression progress, language modeling as compression, active inference, cybernetics, e-graphs, vector-symbolic approaches, and morphological computation.
- It provides definitions, laws, metrics, case studies, failure modes, reviewer objections, and a research agenda.
- No local CGS benchmark, proof, or implementation was run in this repo as part of this note.

## Failure Modes

- Hidden complexity debt and residual explosion.
- Overcompression that destroys necessary context or variation.
- Unverified generation and degenerate governance.
- Recursive instability when compact structures modify the system that defines or evaluates them.
- Treating compactness as automatically interpretable, safe, or superior to scale.

## Claims To Add Or Update

- CGS can support source-derived claims about residual honesty, generate-verify-repair compression, compact seeds, and governance interfaces after chapter-specific mappings are made.
- Use CGS to frame fast generation as compact generative leverage only when residual burden, verification cost, fallback, and governance interface remain visible.
- It should not be used to claim that compactness by itself proves safety, interpretability, or adequacy.
