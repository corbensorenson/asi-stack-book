## Reader Promise

This pass should let a human reader understand why fast generation is credible
only when proposed output, accepted output, verifier cost, fallback, memory
pressure, repair, and task success are counted separately.

The reader throughline is:

> speed only counts after acceptance, verification, fallback, and memory
> pressure are paid.
