# Round 15 Proof-Depth Disposition

Date: 2026-07-18  
Disposition: **the four named modules are retained as finite semantic refinements, not counted as evidence for their chapter-core claims**

The review's `91 derived / 278 projection` snapshot no longer describes the
repository. The frozen rationalization baseline currently classifies 952 of
1,151 declarations as derived/decomposed, 187 as direct/projection, and 12 as
unknown/mixed. Those are audit classes, not scientific accomplishments.

More importantly, the four modules named by the review are not empty stubs:

| Module | Current semantic surface | General derived result added in this cycle | Evidence boundary |
|---|---|---|---|
| `DataEngineLifecycleRefinement` | 8 stages, 82 routes, full-state/update/deletion/custody composition, 96 route mutations and 3 cross-stage mutations | Any rejected event preserves the complete state; any accepted event increments receipt custody | No data quality, learning, forgetting, erasure, privacy, rollback-efficacy, or deployment result |
| `OpenEndedImprovementRefinement` | 7 stages, 81 routes, archive/evaluation/adjudication/governor handoff, 91 route mutations and 3 cross-stage mutations | Any rejected event preserves the complete state; any accepted event increments receipt custody | No open-ended generator, semantic novelty, usefulness, hazard-control, or self-improvement result |
| `PolicyOptimizationRefinement` | 7 stages, 63 routes, full training-state/update/evaluation/lease/readmission, 73 route mutations and 3 cross-stage mutations | Any rejected event preserves the complete state; any accepted event increments receipt custody | No optimizer, model update, causal improvement, forgetting-control, or policy-quality result |
| `ResourceEconomicsRefinement` | 9 stages, 66 routes, budget/reservation/scheduling/verification/transfer/reconciliation closure, 57 rejecting route cases | For every state, event, and packet, a support-promotion or external-effect request can never be an accepted route | No economic optimality, useful-throughput, deployed scheduling, simulation adequacy, or savings result |

The new theorems are quantified over arbitrary states, events, and packets.
They are not restatements of a concrete fixture hypothesis. The first three are
proved from `applyEvent` semantics; the fourth performs case analysis across
the route's precedence gates and authority booleans. The existing full-cycle
theorems remain executable finite proofs and are retained, but theorem count or
`native_decide` alone is never treated as depth.

All four modules compile independently and as part of `lake build`. Their
Python consumers rerun inherited result validators, digest-bind exact source
artifacts, exercise every declared route, preserve failed prefixes, and reject
cross-stage continuation. This disposition closes the specific “execute or
declare” criticism. It does not close the roadmap's wider semantic-formal-depth
milestone, which still owns other consequential modules and any claim that
needs a richer mathematical model rather than a finite policy machine.
