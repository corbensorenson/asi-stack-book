# Post-v2 Empirical Residual Ledger

Recorded: 2026-07-10

This ledger preserves what the three completed empirical programs did not
close. A completed run is not a discharged residual.

| Owner | Residual | State | Reopen condition |
|---|---|---|---|
| Governed work | Two deliberately sabotaged rollback cases retain unauthorized-path residuals after inexact rollback. | open and quarantined | A stronger rollback service removes and independently verifies all effect surfaces without erasing the failed records. |
| Governed work | The governed route had zero useful releases on 16 plan-conditioned candidates. | open usefulness residual | Broader natural workloads and models measure useful throughput, unsafe release, refusal calibration, and governance cost together. |
| Governed work | Observer separation is process/implementation separation inside one project. | open independence residual | Independent verifier-quality or external institutional evidence exists; external human review is not required for this roadmap. |
| Routing | Oracle, learned, rule, and fallback routes were indistinguishable because request features were too separable. | open validity residual | An ambiguous held-out workload separates routing policies without label leakage. |
| Routing | Fallback and abstention activated zero times. | open coverage residual | A frozen workload exercises both and measures calibration, utility, and safety. |
| Deliberation | The verifier is deterministic task recomputation and candidates are not language-model reasoning traces. | open transfer residual | A language-model workload with independently assessed verifier behavior reproduces benefit and harm. |
| Deliberation | Fixed three-step computation harmed 15 initially correct answers. | open stopping-policy residual | A stronger stopping policy preserves these negative cases and reproduces harm avoidance under less scripted candidates. |
| Update causality | Bounded and regularized updates reduced retained-base accuracy on average. | open forgetting residual | A governed update improves target utility without hidden retained-task loss across seeds. |
| Update causality | Best/final checkpoints disagreed on 62 test decisions. | open authority residual | Checkpoint authority is selected prospectively and survives independent replay without best/final substitution. |
| Update causality | Deletion-aware retraining changed a poisoned cohort but did not prove influence or storage erasure. | open unlearning residual | Descendant and storage propagation, privacy, member/nonmember behavior, and independently replayed deletion are verified. |
| Update causality | Rollback identity covers recorded model tensors, not optimizer state, caches, backups, or external systems. | open rollback-scope residual | A complete state inventory and external effect replay establish exact rollback scope. |
| Conditional lane | Model-weight custody and hardware roots of trust lack a disposable attestation environment. | deferred; activation absent | Non-secret measured-boot/device-binding/rotation/revocation environment is registered. |
| Conditional lane | Personal compute hives lack a distinct-principal multi-node environment. | deferred; activation absent | A disposable multi-node distinct-principal environment is registered. |
| Conditional lane | Circle/Coil lacks a trainable matched model-quality implementation. | deferred; activation absent | A trainable implementation and matched baseline are registered. |

These residuals block promotion but do not make the completed experiments
failures. Negative and zero-use outcomes are part of the evidence packet.
