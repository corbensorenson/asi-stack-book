# Reader Artifact Layout Review

Last updated: 2026-06-28

This note records a representative local layout spot check for ignored
human-reader artifacts. It is not a full reader manuscript review, not a full
page-by-page PDF review, not an EPUB/DOCX acceptance pass, and not an edition
release record.

## Inputs

The review used local ignored artifacts generated by the reader-format dry-run
workflow:

- HTML snapshot: `build/reader_edition/format_artifacts/html/_reader_site/`
- PDF snapshot: `build/reader_edition_pdf_probe_utf8/format_artifacts/pdf/_reader_site/The-ASI-Stack.pdf`
- Local PDF metadata: `pdfinfo`
- Local PDF text extraction: `pdftotext`
- Local PDF page images: `pdftoppm`
- Local browser screenshots against the preserved HTML snapshot

## Fix During Review

The first HTML browser probe exposed a preservation bug rather than a manuscript
layout problem: the dry-run snapshot copied `.html` files but not the rendered
`_reader_site` dependencies. File-based HTML screenshots therefore loaded
without the Quarto CSS and assets, producing raw navigation and mobile horizontal
overflow.

`scripts/render_reader_formats.py` now snapshots the complete `_reader_site`
tree for HTML. After regenerating the reader dry run, the HTML snapshot included
site assets and styles:

| Probe | Viewport | Scroll width | Client width | Horizontal overflow | Stylesheets |
|---|---:|---:|---:|---:|---:|
| index desktop | 1366 x 900 | 1366 | 1366 | no | 7 |
| opening chapter mobile | 390 x 844 | 390 | 390 | no | 10 |
| external sources mobile | 390 x 844 | 390 | 390 | no | 7 |

PDF. Local text extraction found the book title, `Reader Edition Draft`, and
compact `evidence boundary: architectural argument` text.

|---|---|

## Residuals

- This is not a full chapter-by-chapter reader manuscript review.
- This is not a full 574-page PDF inspection.
- EPUB was structurally inspected but not opened in an e-reader application.
- DOCX was structurally inspected but not opened in Word, LibreOffice, or Google
  Docs.
- Dense diagrams, long tables, bibliography behavior, search behavior, and
  cross-reference behavior still need broader manual review.
- No artifact should be called release-ready until a reviewed edition release
  record names it and records residuals.
