# Round 18 Bounded Breadth-Completion Adjudication

Date: 2026-07-24
Decision owner: Corben Sorenson
Evidence effect: none; manuscript and roadmap integration only
Terminal state: admitted breadth packet, followed by a structural freeze

## Decision

Round 18 temporarily superseded the earlier rule that no chapter admission could
precede P2 materialization. The owner's explicit strategy was to finish the
book's conceptual map first, put the prose in the manuscript, and then return to
empirical work without leaving architectural ideas suspended in planning.

This amendment authorizes one bounded completeness packet. It does not declare
P2 solved, excuse the exact materialization attempt, reopen evidence support, or
create a standing license for more chapter growth. Once this packet is terminal,
the chapter count freezes again and the first empirical action is a commit-bound
P2 capacity measurement plus either an exact protocol attempt or a durable
failure receipt naming the command, exit state, and blocker.

## Admission test

A proposed chapter passed only when all five conditions held:

1. it owned a system boundary or failure class not already owned by a manifest
   chapter;
2. routing it into an existing chapter would make that chapter carry a second
   independent thesis;
3. the topic had a distinct mechanism, interface, invariant, failure-mode, and
   evaluation surface;
4. primary-source comparators could be attached without treating literature as
   local evidence; and
5. the new chapter could state its non-claims and first discriminating test.

## Rejected chapter shapes

- No generic “alternative architectures” extension was admitted here; the
  replaceable-substrates chapter already owns architecture substitution.
- No standalone RAG chapter was admitted; retrieval is one governed context
  acquisition mechanism.
- No standalone uncertainty chapter was admitted; uncertainty changes the
  world-model/evidence interface and is meaningful only against decisions.
- No standalone scaling or emergence chapter was admitted; the efficient-ASI
  chapter owns compute/capability allocation and forecasting limits.
- No standalone decision-theory survey was admitted; the useful system boundary
  is how a selected rule affects commitments, delegation, and interaction.

## Birth obligations

The five new chapters must ship atomically with:

- manifest entries and Quarto ordering;
- full prose, Human Reading Paths, Mermaid diagrams, and explicit non-claims;
- source inventory records, notes, and claim-source mappings;
- one birth claim atom per chapter;
- a reader-order and handoff projection;
- a copied-scaffold/inheritance check recorded in
  `evidence_quality/round_18_breadth_completion_inheritance_audit.json`;
- changelog and roadmap reconciliation; and
- local validation and full render.

The packet does not retroactively close the six-chapter Round 16 atom debt, the
W3 whole-book repetition guard, the White-Box depth amendment, or the current
reader-freshness packet. It is constructed so those debts do not increase.

## Breadth stop rule

After this packet, breadth is frozen. One post-tranche completeness sweep may
classify a newly discovered concern, but the default outcome is routing to an
existing owner. A future chapter requires a dated amendment showing that the
five-part admission test still passes and identifying which completed empirical
or evidence milestone makes the addition worth the integration cost.

The present conceptual map is considered complete enough to begin depth work
when ordinary new ideas can be assigned to an existing chapter without giving
that owner a second thesis. “Interesting” is not an admission criterion.

## First actions after terminal admission

1. Remeasure P2 capacity and Docker state at run start.
2. Execute the frozen P2 materialization command, or record the exact attempted
   command, exit state, and blocker without substituting more prose.
3. Continue the six-chapter Round 16 atom pack.
4. Run W3 over the full current manifest and add the prospective inheritance
   guard.
5. Close the White-Box depth amendment.
6. Derive the current-reader freshness packet.

This ordering preserves the owner's breadth-first decision while accepting the
Round 18 criticism that an “empirical headline” is meaningless if it never
receives an exact attempt.
