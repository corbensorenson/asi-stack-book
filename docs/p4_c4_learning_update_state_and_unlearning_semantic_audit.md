## The decisive semantic split

Behavioral cohort change is not causal influence removal. Influence removal is
not privacy-risk reduction. Lineage invalidation is not primary-storage
erasure, and primary-storage erasure is not backup, replica, regeneration, or
remote-descendant erasure. None of those technical axes decides legal
compliance. The Data Engine formalization preserves the split; Weight Custody
and Context Transactions bound adjacent storage and transaction records without
silently completing it.

Likewise, “full state” is inventory-relative. The finite modules name model,
optimizer, scheduler, RNG, cache, checkpoint, backup, descendant, evaluator,
authority, and receipt surfaces, but cannot prove the inventory complete. A
digest match proves equality over declared bytes or records, not restoration of
unobserved state or reversal of external effects.

## Residual ownership

Data Engines owns causal forgetting, privacy, deletion propagation, storage and
backup closure, and natural-task learning evidence. Policy Optimization owns
reward validity, actual optimization, causal update value, monitoring, and live
rollback. Weight Custody owns authentic cryptographic/hardware enforcement and
copy/recipient closure. Context Transactions owns real concurrency, durability,
recovery, and cache invalidation behavior. Those burdens remain empirical and
operational; this formal cluster contributes no support, release, or publication
movement.
